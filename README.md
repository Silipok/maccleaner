## Обзор

**MacCleaner** — это легковесная CLI-утилита для macOS, которая помогает разработчикам освободить десятки гигабайт дискового пространства за считанные секунды. Она находит и удаляет:

- Кэши приложений (`~/Library/Caches`)
- Логи (`~/Library/Logs`)
- Содержимое Корзины (`~/.Trash`)
- Артефакты Xcode (DerivedData, Archives, DeviceSupport, CoreSimulator)
- Кэши менеджеров пакетов (Homebrew, CocoaPods, Yarn, NPM, Gradle)

Утилита использует высокопроизводительное **параллельное сканирование** файловой системы и **macOS-специфичные оптимизации**, вдохновлённые проектом [Spacie](https://github.com/AlexGladkov/Spacie).

| 🧹 **Очистка по категориям** | Удаление системных кэшей, артефактов Xcode или dev-инструментов |

- **macOS** (поддержка APFS и HFS+)

- **Cargo** (поставляется с Rust)

## Установка и сборка

### 1. Клонирование репозитория

```bash
git clone <your-repo-url>
cd maccleaner
```

### 2. Сборка

```bash
# Debug-сборка (быстрая компиляция, для разработки)
cargo build

# Release-сборка (оптимизированная, для использования)
cargo build --release
```

### 3. Запуск

```bash
# Прямой запуск через Cargo
cargo run

# Или запуск скомпилированного бинарника
./target/release/maccleaner
```

### 4. Обновление Rust (при необходимости)

```bash
rustup update
```

### Режимы работы

#### 1. Interactive Clean (Интерактивная очистка)

Сканирует **все** категории, затем предлагает checkbox-список, где можно выбрать **конкретные** цели для удаления. Используется библиотека `dialoguer` для интерактивного многовыборного интерфейса.

- `Space` — выбрать/снять выбор
- `Enter` — подтвердить выбор

#### 2. Scan All (Сканирование всего)

Только **сканирует** все категории и выводит таблицу размеров. Ничего не удаляет. Полезно для получения общей картины занятого места.

Пример вывода:
```
Scanning System (Caches, Logs, Trash)...
Category             Size            Description                    Path
----------------------------------------------------------------------------------------------------
User Caches          2.4 GiB         Application caches             ~/Library/Caches
User Logs            51.3 MiB        Application and system logs    ~/Library/Logs
Trash                0 B             Deleted files in Trash         ~/.Trash
----------------------------------------------------------------------------------------------------
Total: 2.4 GiB
```

#### 3. Quick Clean System (Быстрая очистка системы)

Очищает системные кэши, логи и Корзину. Перед удалением показывает, что будет удалено, и запрашивает подтверждение `(y/N)`.

#### 4. Clean Xcode (Очистка Xcode)

Удаляет артефакты Xcode: DerivedData, Archives, iOS/watchOS DeviceSupport, CoreSimulator.

#### 5. Clean Developer Tools (Очистка инструментов разработчика)

Очищает кэши пакетных менеджеров: Homebrew, CocoaPods, Yarn, NPM, Gradle.

#### 6. Run as Administrator (Запуск от администратора)

Перезапускает утилиту с `sudo` для доступа к защищённым системным директориям. При запуске от `root` этот пункт заменяется на `Exit`.

### Описание модулей

#### `main.rs` — Точка входа

Содержит функцию `main()` с бесконечным циклом меню: очистка экрана → баннер → меню → чтение ввода → вызов соответствующей функции. Также содержит функции-обёртки для каждого режима работы:

| Функция | Описание |
|---|---|
| `scan_all()` | Сканирует все 3 категории и выводит общий итог |
| `interactive_clean()` | Собирает все цели, показывает интерактивный выбор, очищает выбранные |
| `clean_system()` | Быстрая очистка системных кэшей, логов, корзины |
| `clean_xcode()` | Очистка артефактов Xcode |
| `clean_dev()` | Очистка кэшей dev-инструментов |

#### `targets.rs` — Определение целей очистки

Определяет структуру `CleanTarget` и фабричные функции для создания списков целей.

**Структура `CleanTarget`:**

```rust
pub struct CleanTarget {
    pub category: String,      // Категория (System, Xcode, Developer Tools)
    pub name: String,          // Человекочитаемое имя
    pub path: PathBuf,         // Абсолютный путь к директории
    pub description: String,   // Описание для пользователя
}
```

**Фабричные функции:**

| Функция | Возвращаемые цели |
|---|---|
| `get_general_targets()` | User Caches, User Logs, Trash |
| `get_xcode_targets()` | Archives, DerivedData, iOS/watchOS DeviceSupport, CoreSimulator |
| `get_dev_targets()` | Homebrew, CocoaPods, Yarn, NPM, Gradle кэши |

#### `cleaner.rs` — Сканирование и удаление

Реализует методы `CleanTarget` и публичные функции для пакетной обработки целей.

**Методы `CleanTarget`:**

| Метод | Описание |
|---|---|
| `size(&self) -> u64` | Возвращает физический размер директории в байтах (через `calculate_dir_size`) |
| `clean(&self) -> io::Result<u64>` | Удаляет содержимое директории, сохраняя саму папку. Возвращает количество освобождённых байт |

**Публичные функции:**

| Функция | Описание |
|---|---|
| `scan_targets(&[CleanTarget], &str) -> u64` | Сканирует список целей, выводит таблицу размеров, возвращает общий размер |
| `clean_targets(&[CleanTarget], &str)` | Показывает что будет удалено, запрашивает подтверждение `(y/N)`, выполняет удаление и выводит итог |

**Стратегия удаления в `clean()`:**
- Итерирует по **непосредственным** дочерним элементам целевой директории
- Для поддиректорий — `fs::remove_dir_all()`
- Для файлов — `fs::remove_file()`
- **Не удаляет** саму корневую директорию (например, `~/.Trash` или `~/Library/Caches`)
- При ошибке удаления конкретного элемента выводит предупреждение и продолжает работу

#### `ui.rs` — Пользовательский интерфейс

Управляет весь визуальным выводом и пользовательским вводом.

| Функция | Описание |
|---|---|
| `clear_screen()` | Очищает терминал ANSI escape-кодом `\x1B[2J\x1B[1;1H` |
| `print_banner()` | Выводит ASCII-баннер с названием и версией |
| `is_root() -> bool` | Проверяет, запущена ли утилита от `root` (через `libc::geteuid()`) |
| `restart_with_sudo()` | Перезапускает текущий исполняемый файл через `sudo` |
| `print_menu()` | Выводит главное меню (с разным набором пунктов для `root` и обычного пользователя) |
| `wait_for_enter()` | Ждёт нажатия Enter перед возвратом в меню |
| `prompt_clean_targets(&[(&CleanTarget, u64)]) -> Vec<usize>` | Интерактивный multiple-select с размерами целей |

#### `utils.rs` — Утилиты

Содержит низкоуровневые функции для подсчёта размера и определения домашней директории.

| Функция | Описание |
|---|---|
| `calculate_dir_size(&Path) -> u64` | Параллельный подсчёт физического размера директории с дедупликацией hard links |
| `get_home_path() -> PathBuf` | Возвращает домашнюю директорию из переменной окружения `HOME` |

## Технические детали

### Параллельное сканирование (jwalk)

Стандартная библиотека `walkdir` обходит файловое дерево **в одном потоке**, что крайне медленно для директорий с миллионами файлов (например, `DerivedData` или `node_modules`).

MacCleaner использует [`jwalk`](https://crates.io/crates/jwalk) — drop-in замену для `walkdir`, работающую поверх `rayon`. Она автоматически распределяет обход поддиректорий по **нескольким потокам**, что позволяет максимально загрузить SSD параллельными запросами на чтение метаданных.

```rust
use jwalk::WalkDir; // Вместо walkdir::WalkDir

WalkDir::new(path)
    .into_iter()
    .filter_map(|entry| entry.ok())
    // ... обработка
```

**Результат:** директории с десятками тысяч файлов сканируются в разы быстрее.

### Физический размер файлов (blocks)

На APFS (файловая система macOS) логический размер файла (`metadata.len()` / `st_size`) может **не совпадать** с реально занятым местом на диске:

- **Sparse-файлы** — логически «большие», но физически занимают мало места
- **APFS Compression** — файлы с флагом `UF_COMPRESSED` прозрачно сжимаются ОС
- **Copy-on-Write** — клонированные файлы могут разделять физические блоки

MacCleaner считает **физический** размер через `st_blocks`:

```rust
use std::os::unix::fs::MetadataExt;

// Физический размер = количество 512-байтных блоков × 512
let physical_size = metadata.blocks() * 512;
```

Это даёт такой же результат, как команда `du -s`, и точно отражает, сколько места **реально будет освобождено** при удалении.

### Дедупликация жёстких ссылок (Hard Link Deduplication)

macOS активно использует **жёсткие ссылки** (hard links), особенно в:
- Бэкапах Time Machine
- Кэшах системных фреймворков
- Некоторых приложениях (например, Xcode SDK)

Жёсткая ссылка — это несколько записей в файловой системе, указывающих на **один и тот же** физический файл (inode). Без дедупликации сканер посчитает размер такого файла **несколько раз**.

MacCleaner решает эту проблему, отслеживая inode файлов с `nlink > 1`:

```rust
let nlink = metadata.nlink(); // количество жёстких ссылок
let inode = metadata.ino();   // уникальный идентификатор файла

if nlink > 1 {
    let mut seen = seen_inodes.lock().unwrap();
    if !seen.insert(inode) {
        return 0; // Этот файл уже посчитан через другую ссылку
    }
}
```

`HashSet` обёрнут в `Mutex`, поскольку `jwalk` обходит файловое дерево в нескольких потоках одновременно.

### Стратегия безопасного удаления

1. **Никогда не удаляет корневые директории** — удаляется только **содержимое** таких папок, как `~/.Trash`, `~/Library/Caches` и т.д. Сами папки остаются нетронутыми.
2. **Подтверждение перед удалением** — перед любой деструктивной операцией утилита показывает, что будет удалено, и запрашивает явное подтверждение `(y/N)`.
3. **Graceful error handling** — при ошибке удаления конкретного файла/директории (например, заблокированного процессом) утилита выводит предупреждение и продолжает работу с остальными элементами.

## Запуск от имени администратора

Некоторые системные директории (например, `/Library/Caches`) требуют прав `root`. Для доступа к ним:

1. Выберите пункт **6. Run as Administrator (sudo)** в меню
2. Введите пароль администратора
3. Утилита перезапустится с расширенными правами

Или запустите напрямую:
```bash
sudo ./target/release/maccleaner
```

> ⚠️ **Внимание:** при запуске от `root` утилита получает доступ ко **всем** файлам системы. Будьте внимательны при выборе целей для очистки.
